# inVision U — Финальный README

## О проекте

**inVision U** — это современный AI-ассистированный портал отбора, созданный для красивой, уверенной и технологичной работы с заявками абитуриентов.
также у нас есть деплой сайта: https://invisiionu.netlify.app , COMMITTEE_PASSWORD=m1m2m3m4m5m6m4m2m6

Проект объединяет:
- **интуитивный кандидатский путь** — от заполнения анкеты до отправки заявки;
- **умную панель комиссии** — для просмотра, анализа и сравнения кандидатов;
- **AI-поддержку принятия решений** — с понятными evidence-блоками, оценками по критериям и наглядной аналитикой.

inVision U задуман как цельная digital-система, где подача заявки ощущается премиально и понятно, а работа комиссии становится быстрее, прозрачнее и удобнее.

---

## Ключевые преимущества

### Для кандидата
- пошаговая, логичная и приятная форма подачи заявки;
- сбор персональных данных, достижений, активностей и академической информации;
- отдельный шаг для эссе;
- отдельный шаг для интервью-ответов;
- поддержка голосового ввода;
- подтверждение согласия прямо внутри сценария подачи;
- аккуратный финальный экран отправки.

### Для комиссии
- быстрый доступ к заявкам через отдельный интерфейс;
- список кандидатов со статусами и ключевыми метриками;
- детальная карточка каждой заявки;
- AI-анализ по основным критериям;
- evidence-подход с цитатами и пояснениями;
- compare view для двух кандидатов;
- аналитический экран для общего обзора;
- decision support вместо “черного ящика”.

inVision U не просто хранит заявки, а превращает процесс отбора в **ясный, красивый и аргументированный workflow**:

- evidence-блоки дают основания для обсуждения;

## Архитектура проекта

### 1. Кандидатский сценарий
Кандидат проходит последовательный flow:

1. персональные данные;
2. академический и внеучебный бэкграунд;
3. эссе;
4. интервью;
5. согласие и отправка.

Это создаёт аккуратный и понятный пользовательский опыт без перегрузки.

### 2. Серверная обработка заявки
После отправки данные сохраняются в базе и запускается AI-пайплайн оценки, который:
- собирает сигналы из анкеты, эссе и интервью;
- формирует скоринг по критериям;
- подготавливает evidence;
- сохраняет результат для дальнейшей работы комиссии.

### 3. Интерфейс комиссии
Панель комиссии даёт удобный набор инструментов:
- просмотр списка заявок;
- открытие детальной карточки;
- чтение AI-анализа;
- просмотр оснований оценки;
- сравнение двух кандидатов;
- обзор общей аналитики.

---

## AI-логика проекта

AI-слой в inVision U используется как **инструмент поддержки решений**.

Система помогает комиссии быстро увидеть:
- мотивацию кандидата;
- опыт;
- лидерские сигналы;
- потенциал роста;
- дополнительные сильные стороны и зоны внимания.

Особенно важная часть — **evidence-подход**:
каждая оценка подкрепляется понятными основаниями, чтобы анализ был не абстрактным, а аргументированным и наглядным.

## Переменные окружения

Создайте `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
OPENAI_API_KEY=your_openai_api_key
COMMITTEE_PASSWORD=your_committee_password
```

### Назначение переменных
- `NEXT_PUBLIC_SUPABASE_URL` — адрес проекта Supabase;
- `SUPABASE_SERVICE_ROLE_KEY` — серверный ключ для работы API;
- `OPENAI_API_KEY` — ключ для AI-анализа и транскрибации;
- `COMMITTEE_PASSWORD` — пароль доступа в интерфейс комиссии.

## Быстрый запуск

### 1. Установить зависимости
```bash
npm install
```

### 2. Настроить окружение
Создайте `.env.local` и заполните нужные переменные.

### 3. Подготовить базу данных
Откройте Supabase SQL Editor и выполните содержимое файла:

```bash
supabase-schema.sql
```

### 4. Запустить проект
```bash
npm run dev
```

После запуска приложение будет доступно локально по адресу:

```bash
http://localhost:3000
```
